## L1 基础理论(8 题)

### L1.1-1 (选择)
**题目**:Prefill 与 Decode 正确描述?
**选项**:
- A. Prefill 瓶颈是带宽
- B. Prefill 并行算所有 token,Decode 每次算一个新 token
- C. Decode 不需 KV Cache
**答案**:B。1.1 节:Prefill 算力瓶颈,Decode 带宽瓶颈,KV Cache 跨两阶段。

### L1.1-2 (选择)
**题目**:70B 模型 128k 上下文,KV Cache 显存最接近?
**选项**:
- A. <1GB
- B. 2-4GB
- C. 16-24GB
- D. >100GB
**答案**:C。1.1 节公式,70B 128k 上下文显存账 16-24GB。

### L1.2-1 (选择)
**题目**:Token 三角约束,以下错误?
**选项**:
- A. 三者此消彼长
- B. 输出比输入贵 5 倍
- C. 标称 200k 等于有效 200k
- D. Prompt Caching 砍前缀成本 90%
**答案**:C。1.2 节:Lost in the Middle 使中间信息准确率掉 30+ 个百分点。

### L1.3-1 (判断)
**题目**:CoT 在所有任务上加都能提升准确率。
**选项**:
- 对
- 错
**答案**:错。1.3 节:简单任务加 CoT 反而掉 5-15%,仅适合多步推理。

### L1.4-1 (选择)
**题目**:ReAct 在 HotpotQA 准确率高于 CoT 的根本原因?
**选项**:
- A. 模型参数多
- B. Observation 把工具事实喂回模型纠正错误
- C. 不需要 prompt
- D. CoT 不能 few-shot
**答案**:B。1.4 节:ReAct 是 Thought+Action+Observation,Observation 是真值来源。

### L1.5-1 (选择)
**题目**:ReWOO 省 64% token 且准确率高 4 个百分点的前提?
**选项**:
- A. 不需要工具
- B. 工具调用确定、步骤可预测
- C. 模型越大越好
- D. Few-shot 越多越好
**答案**:B。1.5 节:ReWOO 适用边界是"工具调用确定 + 步骤可预测"。

### L1.6-1 (选择)
**题目**:Plan-and-Execute 相比 ReAct 在长任务(5+ 步)的核心优势?
**选项**:
- A. 不需要 LLM
- B. 显式 Plan + Re-Planner 处理失败
- C. 不需要工具
- D. 只能单步推理
**答案**:B。1.6 节:Re-Planner 让长任务成功率显著提升。

### L1.8-1 (判断)
**题目**:LLM 直接算 "1234567 × 8910111" 准确率 30%,写代码再调工具接近 100%。这说明 LLM 几乎所有任务都应直接交给 LLM。
**选项**:
- 对
- 错
**答案**:错。1.8 节:精确计算等是 LLM 弱项(<0.5),应走专用工具。

## L2 上下文工程(10 题)

### L2.1-1 (选择)
**题目**:Lost in the Middle,关键信息在文档中间时,准确率约?
**选项**:
- A. 100%
- B. 70%+
- C. 约 30%
- D. 0%
**答案**:C。2.1 节:U 形曲线,首尾 70%+、中间暴跌到 30%。

### L2.2-1 (选择)
**题目**:RAG 三件套,正确的是?
**选项**:
- A. 纯向量检索就是完整 RAG
- B. BM25 在含专有名词场景常比纯向量准确率高 10-20 个百分点
- C. Cross-Encoder 比 Bi-Encoder 快 100 倍
- D. 注入时关键信息放中间
**答案**:B。2.2 节:BM25 在专有名词语料上常比纯向量强 10-20 个百分点。

### L2.3-1 (选择)
**题目**:Embedding 选型,以下错误?
**选项**:
- A. MTEB 第一不一定适合垂直领域
- B. 英文 SOTA 做中文检索常不如专门中文模型
- C. 垂直"小模型 + 行业微调"可能反超通用大模型
- D. MTEB 分数是最终决策依据
**答案**:D。2.3 节:"MTEB 分数只能做参考,不是最终决策依据"。

### L2.4-1 (判断)
**题目**:生产里 80% 的 RAG 业务用 pgvector 就够了,数据量过 1 亿或需要毫秒级延迟时才考虑 Milvus / Qdrant。
**选项**:
- 对
- 错
**答案**:对。2.4 节:pgvector 在 10M 以下 SQL 一把梭,1M 向量 P95 延迟约 20ms。

### L2.5-1 (选择)
**题目**:GraphRAG 什么场景下反而"负优化"?
**选项**:
- A. 任何场景
- B. 实体关系稀疏时(<10% 文档含实体关系)
- C. 数据量小于 100 条
- D. 用 GPT-3
**答案**:B。2.5 节:关系密度<10% 时 GraphRAG 准确率比向量 RAG 低 5-10 个百分点。

### L2.6-1 (选择)
**题目**:LangChain `ConversationSummaryBufferMemory` 的核心思想?
**选项**:
- A. 保留所有历史
- B. 只用全量摘要
- C. 滑动窗口 + 关键事件 + 历史摘要的混合策略
- D. 不保留任何历史
**答案**:C。2.6 节:工业级标配是"滑动窗口 + 关键事件 + 摘要"分层。

### L2.7-1 (选择)
**题目**:MemGPT 记忆分几层?
**选项**:
- A. 1 层
- B. 2 层
- C. 3 层:Core + Recall + Archival
- D. 5 层
**答案**:C。2.7 节:Core(约 2k token 永远在 prompt)、Recall(向量召回 session)、Archival(跨 session)。

### L2.8-1 (判断)
**题目**:LLMLingua 压缩 50% token 后 GSM8K 准确率反而升 2-3 个百分点,等价于"反噪声 + 反 Lost in the Middle"。
**选项**:
- 对
- 错
**答案**:对。2.8 节:压缩 50% 后准确率反升 2-3%,原因:去噪、关键信息集中甜区。

### L2.9-1 (选择)
**题目**:Anthropic Prompt Cache 命中后,以下哪项不是收益?
**选项**:
- A. 输入成本降 90%
- B. 首 token 时间大幅下降
- C. 缓存命中必然提升回答语义质量
- D. 用户感知"秒开"
**答案**:C。2.9 节:缓存价值是"降成本 + 降延迟",语义质量不变。

### L2.10-1 (选择)
**题目**:以下哪条不是 L2 上下文注入的常见反模式?
**选项**:
- A. System Prompt 塞 1000 行业务规则
- B. RAG 检索结果加 Cross-Encoder 重排
- C. 整段对话历史无脑塞
- D. 忽略 Prompt Injection 防护
**答案**:B。2.10 节:"加 Cross-Encoder 重排"是 RAG 最佳实践(2.2 节)。

## L3 协议(10 题)

### L3.1-1 (选择)
**题目**:OpenAI Function Calling 与 Anthropic Tool Use 差异,以下错误?
**选项**:
- A. OpenAI 用 `function.parameters`,Anthropic 用 `input_schema`
- B. Anthropic 工具失败返回 `is_error: true`
- C. OpenAI `tool_choice: "required"` 协议级强制必调
- D. 两协议完全一致可直接互转
**答案**:D。3.1 节:两协议字段名与响应类型都不同,需写 adapter。

### L3.2-1 (选择)
**题目**:工具 JSON Schema 中,对 LLM 选对工具准确率影响最大的是?
**选项**:
- A. 字段 `type` 类型
- B. 字段 `enum` 范围
- C. 工具和字段的 `description`(信息密度)
- D. `additionalProperties: false`
**答案**:C。3.2 节:description 从 10 词提到 50 词,选对率从 60% 涨到 92%。

### L3.3-1 (判断)
**题目**:MCP 四大原语含 Sampling,允许 MCP Server 主动向 Host 请求 LLM 采样,这是 Function Calling 完全做不到的双向通信能力。
**选项**:
- 对
- 错
**答案**:对。3.3 节:Sampling 是 MCP 独有的双向通信能力。

### L3.4-1 (选择)
**题目**:MCP Server 实战中,80% 工程坑不在协议本身,而是?
**选项**:
- A. JSON Schema 编写
- B. 传输层选型(stdio vs HTTP+SSE)和安全沙箱
- C. Prompt 模板
- D. Few-shot
**答案**:B。3.4 节:80% 工程坑在传输层(stdio 本地、HTTP+SSE 生产)和安全沙箱。

### L3.5-1 (选择)
**题目**:A2A 协议与 MCP 的关系?
**选项**:
- A. A2A 取代 MCP
- B. 互补:A2A 解决"Agent↔Agent",MCP 解决"Agent↔工具"
- C. 完全无关
- D. MCP 是 A2A 的子集
**答案**:B。3.5 节:两者互补,MCP 让 Agent 用工具,A2A 让 Agent 找专家。

### L3.6-1 (判断)
**题目**:OpenAI Assistants API 全部由服务端托管,OpenAI 已宣布 2026 年底弃用,新项目应迁移到 Responses API + Agents SDK。
**选项**:
- 对
- 错
**答案**:对。3.6 节:Assistants API 灵活性差,推荐 Responses API+Agents SDK。

### L3.7-1 (选择)
**题目**:Anthropic Prompt Caching 盈亏平衡点(命中率多少以上才赚钱)?
**选项**:
- A. 5%
- B. 22%
- C. 50%
- D. 90%
**答案**:B。3.7 节:公式 `(1-h)*1.25 + h*0.1 = 1`,解得 h≈0.217(约 22%)。

### L3.8-1 (选择)
**题目**:LLM 流式响应(SSE),以下错误?
**选项**:
- A. 流式 TTFT 200-400ms,非流式 = 总完成时间
- B. 流式总耗时比非流式少 5-15%
- C. SSE 格式 `data: {...}\n\n`
- D. 生产 Agent 系统必须开流式
**答案**:B。3.8 节:流式总耗时反而比非流式多 5-15%。

### L3.9-1 (选择)
**题目**:LLM 协议演进 18 个月的关键理念?
**选项**:
- A. 新协议取代旧协议
- B. 协议是"层叠"而非"替代",生产是"协议栈"
- C. 永远用最新
- D. 协议只能选一个
**答案**:B。3.9 节:协议演进是"层叠"而非"替代",生产系统是"协议栈"。

### L3.10-1 (选择)
**题目**:协议选型决策树,以下哪种场景最适合直接用 Function Calling?
**选项**:
- A. 20+ 工具 + 跨进程 + 多 Agent
- B. ≤ 5 个工具 + 单进程
- C. 需要 Agent 间互操作
- D. 需要 HTTP+SSE 远程服务
**答案**:B。3.10 节:80% 场景选 Function Calling,典型是 ≤ 5 个工具 + 单进程 + 单 Agent。
